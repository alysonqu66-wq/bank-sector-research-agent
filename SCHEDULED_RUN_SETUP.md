# 定时自动运行配置指南(Windows 任务计划程序)

让 `daily_run.bat` 在每天固定时间自动跑,实现「**每天早上自动拿到一份宏观快报**」。

---

## 一次性配置(~5 分钟)

### Step 1:先确认 .bat 能手动跑通

双击项目根目录的 `daily_run.bat`,看是否能完整执行(会弹出黑窗口)。

预期看到:
```
[当前时间] === 任务开始 ===
[当前时间] [1/3] 拉取数据 ...
   (社融失败正常,只要其他 OK 就行)
[当前时间] [2/3] 合并主表 ...
   主表已保存 ...
[当前时间] [3/3] 生成宏观快报 ...
   快报正文 ...
[当前时间] === 任务完成 ===
```

⚠️ **如果失败,先把单步问题修了再调度** —— 调度本身只是定时执行,不会自己修 bug。

如果想让窗口停住看输出,把 `daily_run.bat` 最后一行的 `REM pause` 改成 `pause`。

---

### Step 2:打开任务计划程序

- 按 `Win` 键 → 搜索「**任务计划程序**」(英文版叫 `Task Scheduler`)→ 打开
- 右侧栏 → 点 **「创建基本任务...」**

---

### Step 3:配置任务

按向导填写:

| 字段 | 填什么 |
|---|---|
| **名称** | `银行Agent每日自动` |
| **描述**(可选) | 每个工作日早上自动拉数据 + 生成宏观快报 |
| **触发器** | 选「每天」(或「每周」勾周一到周五) |
| **开始时间** | **8:30**(早盘前一小时,推荐) |
| **操作** | 选「**启动程序**」 |
| **程序或脚本** | `D:\bank-sector-research-agent\daily_run.bat` |
| **起始于(可选)** | `D:\bank-sector-research-agent` |
| **完成时勾「打开此任务的属性对话框」** | 推荐勾上,接下来做 Step 4 |

---

### Step 4:在属性对话框里调一个关键选项

- 切到「**常规**」标签页
- 勾上「**不管用户是否登录都要运行**」
- 勾上「**使用最高权限运行**」
- 保存

这样**即使你没登录电脑,任务也能跑**(只要电脑开机)。

⚠️ **如果你的电脑会自动睡眠**:切到「**条件**」标签页,**取消勾**「只有在交流电源时启动此任务」,并勾上「**唤醒计算机运行此任务**」。

---

### Step 5:验证

回到任务计划程序主窗口:

- 左侧栏 → 「任务计划程序库」 → 找到「**银行Agent每日自动**」
- 右键 → 「**运行**」
- 等 30 秒,然后看 `outputs/` 是否多了一份 `macro_flash_<时间戳>.txt`

✅ **多了就大功告成**,以后每天 8:30 自动跑。
❌ **没多**:看任务计划程序「上次运行结果」那一列的报错代码,Google 一下就知道原因。

## 常见坑

| 现象 | 原因 | 解决 |
|---|---|---|
| 任务运行但没产出 | 工作目录不对 | 确认 .bat 第一行有 `cd /d D:\bank-sector-research-agent` |
| 提示「找不到 python」 | 虚拟环境没激活 | 确认 .bat 里有 `call .venv\Scripts\activate.bat`,并且 `.venv` 文件夹真存在 |
| 提示「找不到 DEEPSEEK_API_KEY」 | `.env` 路径不对 | 确认 `D:\bank-sector-research-agent\.env` 存在且填了真实 key |
| 任务窗口一闪而过 | 跑太快或报错没看到 | 临时把 `daily_run.bat` 末尾 `REM pause` 改成 `pause` 看输出,调通了再改回去 |
| 电脑睡眠时没跑 | 节能策略 | Step 4 里勾「唤醒计算机运行此任务」 |
| 跑成功但没 wifi | akshare 拉数据失败 | 这是预期 —— `fetch_data` 失败不影响后续,主表会用之前的数据;社融本身就是手动导的,日常 fetch 失败也无所谓 |

## 进阶玩法

### A. 跑完发邮件到自己邮箱

在 `daily_run.bat` 末尾加一行:`python agent\email_notify.py`

然后写一个 `agent/email_notify.py`:
- 读 `outputs/` 下最新的 `macro_flash_*.txt`
- 用 Python 自带 `smtplib` 发邮件
- QQ / 163 / Gmail 都行,搜「Python smtplib QQ 邮箱发邮件」有教程
- 邮箱密码也放 `.env`,不要写代码里

这样早上一睁眼就能在手机上看到当日快报,**这是真正「自动晨报」的体验**。

### B. 跑完发微信(用企业微信群机器人)

如果有企业微信群:
- 群里添加一个机器人,拿到 webhook URL
- Python 一段 requests.post 把快报内容发过去

### C. 跑完自动 push 到 GitHub

如果项目已 push 到 GitHub,在 .bat 末尾加:
```
git add outputs\
git commit -m "auto: daily report %date%"
git push
```

每天自动归档到 GitHub,形成可追溯的运行日志,方便回溯任意一天的产出。

## 不想用 Windows 任务计划程序?
